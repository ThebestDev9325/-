참을인 v1 ~ v30 변경 이력
작성 기준일: 2026-07-18

이 문서는 참을인 Flutter 프로젝트의 Git 커밋, Pull Request, 기존 요구사항 문서를 기준으로 정리했다.
버전 표기의 v숫자는 Android versionCode(빌드 번호)를 우선 기준으로 삼았다.
v2와 v4~v6은 Git에 별도 빌드 번호가 남아 있지 않은 중간 개발 단계이므로 당시 작업 흐름을 기준으로 정리했다.

────────────────────────────────────────
v1 · 1.0.0+1 · 최초 Flutter MVP
────────────────────────────────────────
- 참을인의 기본 Flutter 앱 구조를 만들었다.
- 홈, 글쓰기, 내 기록, 공감, 내 공유, 긍정, 설정으로 이어지는 기본 화면 흐름을 구성했다.
- 감정과 카테고리를 선택하고 직접 참을 인(忍)을 써보는 글쓰기 경험을 구현했다.
- 획순 이미지 7장과 초기 긍정 콘텐츠, 위로 이야기 데이터를 추가했다.
- 기록과 공유 글을 표현할 기본 데이터 모델을 만들었다.

────────────────────────────────────────
v2 · 중간 개발 단계 · 획순 미리보기
────────────────────────────────────────
- 참을 인 획순 이미지가 순서대로 보이는 웹 미리보기 페이지를 만들었다.
- 획순 이미지의 중앙 정렬과 단계별 전환 모습을 점검했다.
- 정식 Android 빌드 번호는 v1과 동일하게 유지됐다.

────────────────────────────────────────
v3 · 1.1.0+3 · Firebase 및 Android 출시 기반
────────────────────────────────────────
- Android 네이티브 프로젝트와 Gradle 빌드 구성을 추가했다.
- Firebase 프로젝트, Firebase Auth, Firestore 연결 기반을 마련했다.
- 사용자별 기록 저장과 공유 사연 저장을 위한 서비스와 보안 규칙을 추가했다.
- Android 앱 아이콘, 시작 화면, 서명 가능한 출시 구조를 준비했다.
- Flutter 위젯 테스트 기반을 추가했다.

────────────────────────────────────────
v4 · 중간 개발 단계 · Firebase 저장 흐름 보강
────────────────────────────────────────
- 로컬 중심 MVP에서 서버 연결형 앱으로 넘어가는 저장 흐름을 점검했다.
- 사용자 기록과 공유 데이터가 Firebase 구조에 맞게 동작하도록 연결부를 다듬었다.
- 별도의 정식 versionCode 증가는 없었다.

────────────────────────────────────────
v5 · 중간 개발 단계 · 하단 광고 영역
────────────────────────────────────────
- 하단 내비게이션 아래에 좌우로 분할된 배너 광고 슬롯을 추가했다.
- 광고 영역이 화면 구조를 밀거나 가리지 않도록 레이아웃을 조정했다.
- 광고 슬롯 너비와 배치를 검증하는 테스트를 추가했다.

────────────────────────────────────────
v6 · 중간 개발 단계 · 오디오·광고 통합 준비
────────────────────────────────────────
- Firebase가 연결된 앱 구조에 효과음과 배경음, 광고 사용 흐름을 통합하기 위한 작업을 진행했다.
- 다음 v7에 포함될 오디오 자산과 화면별 재생 구조를 준비했다.
- 별도의 정식 versionCode 증가는 없었다.

────────────────────────────────────────
v7 · 1.4.1+7 · Firebase·오디오·광고 통합
────────────────────────────────────────
- 홈 숲 소리, 붓 소리, 물방울 버튼음, 감정 선택음, 이야기 전환음 등 화면별 오디오를 추가했다.
- 배경음과 효과음을 관리하는 공통 오디오 서비스를 만들었다.
- Firebase 저장 기능과 광고가 활성화된 앱 화면을 하나로 통합했다.
- Firestore 규칙과 사용자 데이터 저장 항목을 보강했다.

────────────────────────────────────────
v8 · 1.5.0+8 · 오디오 및 테스트 광고 유지
────────────────────────────────────────
- 화면을 이동해도 배경음과 테스트 광고 상태가 끊기지 않도록 개선했다.
- 광고가 일정 시간마다 교대되는 동작을 안정화했다.
- 앱 전역에서 오디오 설정과 광고 영역이 일관되게 유지되도록 정리했다.

────────────────────────────────────────
v9 · 1.6.0+9 · 따뜻한 긍정·위로 콘텐츠
────────────────────────────────────────
- 긍정 문장과 위로 이야기의 표현을 더 따뜻하고 자연스러운 한국어로 개편했다.
- 감정 상황별 이야기의 키워드와 카테고리를 보강했다.
- 인용문 출처 문서를 추가하고 콘텐츠 품질 검증 테스트를 만들었다.

────────────────────────────────────────
v10 · 1.7.0+10 · 온보딩·글쓰기 흐름·앱 아이콘
────────────────────────────────────────
- 닉네임 입력과 앱 시작 온보딩 흐름을 개선했다.
- 글쓰기 진행 단계와 저장·공유 선택 화면을 다듬었다.
- 공유하기 전 계정 연동 안내 흐름을 정리했다.
- 참을인 전용 앱 아이콘을 제작해 Android 해상도별 아이콘에 적용했다.

────────────────────────────────────────
v11 · 1.7.1+11 · 요구사항 감사 및 UI 완성도 보강
────────────────────────────────────────
- v10 요구사항을 조사하고 완료 여부를 점검한 감사 문서를 작성했다.
- 최신 Android용 적응형 앱 아이콘을 추가했다.
- 홈, 공유, 공감, 설정 등 주요 화면의 요구사항 누락을 보완했다.
- Firestore 보안 규칙과 UI 요구사항 테스트를 강화했다.

────────────────────────────────────────

────────────────────────────────────────
- Firebase 연결이 일시적으로 불가능해도 닉네임을 입력하고 앱을 시작할 수 있게 했다.
- 네트워크 실패가 온보딩 전체를 막지 않도록 오프라인 시작 선택지를 제공했다.
- 오프라인 닉네임 흐름을 검증하는 테스트를 추가했다.

────────────────────────────────────────
v13 · 1.8.0+13 · 계정 연동 흐름과 고정 앱 구조
────────────────────────────────────────
- 공유 시 계정 연결을 요구하는 전용 화면과 닉네임 설정 흐름을 구현했다.
- 모든 주요 화면에서 하단 내비게이션과 광고 영역이 유지되는 앱 셸 구조로 개편했다.
- 설정에 계정 관련 항목과 음향 설정을 정리했다.
- 화면 이동 중 음악과 광고가 불필요하게 재생성되지 않도록 개선했다.

────────────────────────────────────────
v14 · 1.8.1+14 · 글쓰기·감정 화면 여백 복원
────────────────────────────────────────
- 글쓰기 화면의 답답해진 여백과 캔버스 크기를 복원했다.
- 감정 선택 카드를 2열 2행으로 안정적으로 배치했다.
- 작은 화면에서도 글쓰기 제목과 선택 카드가 겹치지 않도록 레이아웃 테스트를 추가했다.

────────────────────────────────────────
v15 · 1.8.2+15 · 글쓰기·긍정 화면 안정화
────────────────────────────────────────
- 글쓰기 제목을 한 줄로 유지하고 캔버스 바로 위에 고정했다.
- 긍정 이야기의 본문은 스크롤되되 하단 버튼은 고정되도록 개선했다.
- 다양한 화면 높이에서 오버플로가 발생하지 않도록 레이아웃을 안정화했다.

────────────────────────────────────────
v16 · 1.9.0+16 · 카카오 로그인과 Firebase 계정 연결
────────────────────────────────────────
- Kakao Flutter SDK를 연결하고 카카오톡·카카오계정 로그인을 구현했다.
- Cloud Functions에서 카카오 토큰을 검증하고 Firebase 커스텀 토큰을 발급하도록 구성했다.
- 카카오 사용자별로 기록, 공유 글, 닉네임을 관리할 수 있게 했다.
- Android 카카오 OAuth 리디렉션 설정을 추가했다.

────────────────────────────────────────
v17 · 1.9.1+17 · 계정 상태·홈 브랜딩 강화
────────────────────────────────────────
- 설정 화면에 카카오 계정 연결 상태를 표시했다.
- 홈의 참을인 이름과 슬로건의 크기와 색상을 정리해 브랜딩을 강화했다.
- 계정 상태를 가져오는 Firebase 토큰 클레임 처리를 보강했다.

────────────────────────────────────────

- 홈 배경음을 더 짧고 깨끗한 환경음으로 교체했다.
- 화면의 불필요한 광고 문구와 시각 요소를 줄였다.

────────────────────────────────────────
v19 · 1.9.3+19 · 커뮤니티 보호와 계정 관리
────────────────────────────────────────
- 공유 글 신고 기능과 신고 누적 시 자동 삭제 처리를 구현했다.
- 중복 신고와 본인 글 신고를 차단했다.
- 카카오 계정 탈퇴 시 Firebase Auth, 사용자 기록, 공유 글, 닉네임을 함께 삭제하도록 했다.
- 오늘 작성된 공유 글만 오늘의 BEST 후보로 계산하도록 했다.
- 커뮤니티 보호 로직과 신고 상태 표시를 강화했다.

v23 · 1.9.7+23 · 오디오 생명주기·긍정 문장 표시

v29 · 1.9.13+29 · 날짜별 공감 조회·내 공유 날짜
────────────────────────────────────────

- 긍정 하루 제한 팝업의 어색한 문장 줄바꿈을 수정했다.
- 기존 빌드와 구분되는 내부 테스트용 versionCode 29를 적용했다.

────────────────────────────────────────
v30 · 1.9.14+30 · 연월일 선택·맞춤 위로 이야기 300개
────────────────────────────────────────
- 공감 탭의 날짜 칩 나열을 제거했다.
- 제목 오른쪽에서 연도, 월, 일을 각각 선택하는 간결한 드롭다운을 추가했다.
- 선택한 연·월에 따라 28일, 29일, 30일, 31일이 자동 조정되도록 했다.
